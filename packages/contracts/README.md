# @fel/contracts

Frozen API and data contracts for Financial Evidence Lab (ADR-0001).

| Artifact | Purpose |
|---|---|
| `openapi/openapi.yaml` | REST surface (v1), error envelope, idempotency and concurrency conventions, composite reader endpoint (ADR-0005), observable retrieval (ADR-0006), bounded extraction (ADR-0007) |
| `schemas/*.schema.json` | JSON Schema 2020-12 for core data shapes (source spans, facts, claims, citations, jobs, tenant context, composite reader response, query plan, retrieval event/trace, evidence feedback, extraction payload/event) |
| `fixtures/*.json` | Canonical valid examples, validated in CI |
| `src/generated/api.ts` | TypeScript client types generated by `openapi-typescript` — never edit by hand |
| `src/index.ts` | Public entrypoint re-exporting generated types and schema registry |
| `VERSIONING.md` | The frozen versioning rules |

Regenerate the client after any OpenAPI edit:

```sh
pnpm --filter @fel/contracts generate
```

CI runs `check:generated` (drift fails the build) and the contract tests
(`contracts.test.ts`, via the root vitest run).

Consumers: FastAPI models must round-trip these schemas (contract tests on
the Python side arrive with M0-PLATFORM); the web app imports types from
`@fel/contracts`.
